Console Banking & Shopping System

Build a console-based system to manage users, accounts, and a shopping cart.

Each user should have:

  • username (string)
  • balance (number)
  • cart (object storing items and quantities)

Implement functionality to:

  • Add a user with an initial balance.
  • Deposit or withdraw money from the user’s account.
  • Add items to cart, including item name, price, and quantity.
  • Remove or update items in the cart.
  • Checkout: calculate total cart price, check if user has enough balance, deduct it, and empty the cart if successful.

Provide dynamic status messages:

  • If balance is low → "Insufficient balance"
  • After checkout → "USERNAME successfully purchased items for TOTAL_AMOUNT"
  • When adding/removing items → "ITEM_NAME updated to QUANTITY in the cart"

Handle edge cases gracefully:

  • Trying to withdraw more than balance
  • Adding zero or negative quantity
  • Checking out an empty cart

Notes:

Everything should be dynamic using objects and bracket notation where applicable — users and their carts should be manipulable at runtime.
